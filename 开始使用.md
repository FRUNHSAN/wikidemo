## 🚀 快速开始（3步）

### Windows用户

1. **确保Docker Desktop已安装并运行**
   - 如未安装，下载：https://www.docker.com/products/docker-desktop/

2. **双击运行 `deploy.bat`**
   - 或在PowerShell中执行：`.\deploy.bat start`

3. **浏览器访问**
   - http://localhost
   - 或 http://[::1] （IPv6）

### Linux/macOS用户

```bash
# 1. 添加执行权限
chmod +x deploy.sh

# 2. 启动服务
./deploy.sh start

# 3. 浏览器访问
# http://localhost 或 http://[::1]
```

## 🌐 IPv6访问说明

### 本地访问
- IPv4: `http://localhost`
- IPv6: `http://[::1]`

### 局域网其他设备访问

1. 获取本机IPv6地址：
   ```bash
   # Windows
   ipconfig | findstr "IPv6"

   # Linux
   ip -6 addr show

   # macOS
   ifconfig | grep "inet6"
   ```

2. 在其他设备浏览器访问：
   ```
   http://[你的IPv6地址]
   例如: http://[fe80::1234:5678:90ab:cdef]
   ```

## 📝 常用操作

### 查看服务状态
```bash
# Linux/macOS
./deploy.sh status

# Windows

```

```bash

# 查看特定服务日志

```

## 🔧 故障排查

### 问题1: Docker未安装
**解决**: 安装Docker Desktop
- 下载地址: https://www.docker.com/products/docker-desktop/

### 问题2: 端口被占用
**解决**: 编辑 `docker-compose.yml`，修改端口映射
```yaml
ports:
  - "8080:80"  # 将80改为8080或其他可用端口
```

### 问题3: 无法通过IPv6访问
**检查步骤**:
1. 运行IPv6检查工具：
   ```bash
   ./check-ipv6.sh    # Linux/macOS
   check-ipv6.bat     # Windows
   ```

2. 确认系统启用了IPv6
3. 确认防火墙允许80端口
4. 测试本地连接：`curl -g http://[::1]`

### 问题4: 服务启动失败
**解决**: 查看详细日志
```bash
docker compose logs
```
